数据库安全加固

服务器场景：CentOS6.8（用户名：root；密码：123456）

服务器场景操作系统：CentOS6.8

1.进入虚拟机操作系统：CentOS 6.8，登陆数据库（用户名：root；密码： root），查看数据库版本号，将查看数据库版本号的命令作为
flag 提交；
进入mysql数据库 输入select version(); 查看数据库版本 flag:select version();

2.进入虚拟机操作系统：CentOS 6.8，登陆数据库（用户名：root；密码： root），查看数据库版本号，将查询到的数据库版本号作为
flag 提交；
输入上面的命令 就出答案了 flag:5.0.51a-3ubuntu5

3.进入虚拟机操作系统：CentOS 6.8，登陆数据库（用户名：root；密码： root），查看数据库列表，将查看数据库列表的命令作为
flag 提交；
输入 show databases; flag:show databases;

4.进入虚拟机操作系统：CentOS 6.8，登陆数据库（用户名：root；密码： root），进入 mysql 数据库，查看所有用户及权限，
找到可以从任意 IP 地址访 问的用户，将该用户的用户名作为 flag 提交；（如有多个可以从任意 IP 地址访 问的用户，提交形式
为用户名|用户名|用户名|用户名）
输入show tables;查看数据库中的表

输入 select user,host from user where host=’%’; flag:root #因为这是我自己的靶机,所以没有那么多用户

5.进入虚拟机操作系统：CentOS 6.8，登陆数据库（用户名：root；密码： root），进入 mysql 数据库，查看所有用户及权限，
找到可以从任意 IP 地址访 问的用户，使用 drop 命令将该用户删除，将操作命令作为 flag 提交。
接着上题 查看到了用户及其权限
使用drop user ‘root’@’%’; #删除root用户 删除root用户时要清楚 删除了就登陆不上去了 我这里新建了一个用户

select user,password,host from user; 查看一下是否还有root用户 flag:drop user ‘root’@’%’;

6.进入虚拟机操作系统：CentOS 6.8，登陆数据库（用户名：root；密码： root），进入 mysql 数据库，改变默认 mysql 管理员的
名称，将系统的默认管理员 root 改为 admin，防止被列举，将操作命令作为 flag 提交。
为了做题我又新建了一个root用户

输入update user set user=’admin’ where user=’root’; 进行修改
flag:update user set user=’admin’ where user=’root’;

然后在select user,password,host from user; 查看一下是否更改成功
看到root的地方变为admin 更改成功
